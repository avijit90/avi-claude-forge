List recent Claude Code sessions using the same display name as the `claude --resume` picker. The user is on Telegram — keep the reply compact.

## Display-name priority (mirrors the --resume picker)

For each session, the display name is determined in this order. Use the first one that's non-empty:

1. **Custom session name** — set via `/rename` inside the session. Stored in the session metadata.
2. **Conversation summary** — auto-generated by Claude Code, stored in `sessions-index.json` per project.
3. **First user prompt** — the first user message in the `.jsonl` transcript, truncated to 60 chars.

Searching must match against this same display name (not against an arbitrary derivation).

## Read the session metadata

Sessions live under `~/.claude/projects/<encoded-cwd>/<uuid>.jsonl`. Per-project metadata (when present) is at `~/.claude/projects/<encoded-cwd>/sessions-index.json`. Note: the index is sometimes incomplete or missing entirely due to known bugs — always fall back to scanning the `.jsonl` directly when the index doesn't have what you need.

### For each project directory under `~/.claude/projects/`:

```bash
# 1. Try sessions-index.json first
INDEX=~/.claude/projects/<dir>/sessions-index.json
if [ -f "$INDEX" ]; then
  jq -r '.entries[] | "\(.sessionId)\t\(.summary // "")\t\(.messageCount // 0)"' "$INDEX"
fi

# 2. Then list each .jsonl directly to catch any sessions missing from the index
ls -t ~/.claude/projects/<dir>/*.jsonl 2>/dev/null
```

### For each session, extract:

- **UUID** — `.jsonl` filename without extension
- **Display name** — apply the priority chain:
  - Look for a custom name in the `.jsonl` by scanning the first ~10 lines for `"sessionName"`, `"name"`, or `"title"` fields (the field name varies by Claude Code version):
    ```bash
    head -10 <file> | jq -r 'select(.sessionName // .name // .title) | (.sessionName // .name // .title)' 2>/dev/null | head -1
    ```
  - Otherwise the summary from `sessions-index.json` (if present and non-empty)
  - Otherwise the first user message:
    ```bash
    head -50 <file> | jq -r 'select(.message.role == "user" or .role == "user") | (.message.content // .content)' 2>/dev/null | head -1 | cut -c 1-60
    ```
- **Project path** — decode the parent directory name. The encoding replaces every non-alphanumeric char with `-`. Reverse with `tr '-' '/'` for display, but be aware it's lossy. If decoding produces something obviously broken, fall back to the encoded form.
- **Modified** — `stat -f "%m" <file>` then convert to relative time (`4h ago`, `yesterday`, `3d ago`).
- **Turns** — `wc -l < <file>` (rough proxy for size).

## Filter

If `$ARGUMENTS` is non-empty, filter case-insensitively on the **display name** (not the UUID, not the path). Substring match.

## Sort and limit

Sort by modification time, newest first. Show at most 10 results.

## Reply

```
Recent sessions:
1. <display-name> · <project-path> · <relative-time> · <turns> turns · id: <8-char-uuid>
2. ...

Use /resume-rc <name-or-id> to resume.
```

If empty: `No sessions found.` (or `No sessions matching '<term>'.` when filtered).
